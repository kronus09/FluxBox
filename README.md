# FluxBox - 影视Box 多源聚合引擎

> **当前版本：v0.1.16** | [查看更新日志](#-更新日志)

FluxBox 是一个专为 影视Box 设计的多源聚合引擎，支持聚合和解密多个 影视Box 配置源，提供统一的订阅地址。

## ✨ 特性

- **多源聚合**：支持聚合多个 影视Box 配置源
- **自动脱壳**：支持 AES-CBC、AES-ECB、Base64 及图像隐写术解密
- **智能过滤**：支持关键词过滤，自动排除直播、教育等类型站点
- **聚合模式**：支持"最快站点"（可选60-200个）和"全部站点"两种模式
- **多仓配置**：生成多仓配置，按响应速度排序，可在影视Box中自由切换源
- **健康检查**：自动检测源的健康状态，排除不健康的源，确保聚合质量
- **jar 缓存**：自动缓存 jar 文件，支持 MD5 校验和备份
- **首页菜单选择**：可选择使用哪个源的菜单作为首页显示
- **计划任务**：支持定时自动聚合，可设置每天或每周执行
- **全局状态栏**：标题栏统一显示操作状态，告别突兀的弹窗提示
- **源导入导出**：一键备份和恢复源列表，数据安全有保障
- **源搜索筛选**：支持按名称搜索源，按健康状态/本地化状态筛选
- **Web 管理**：提供简洁的 Web 界面进行管理
- **Docker 支持**：支持 Docker 部署，数据持久化
- **多架构支持**：支持 amd64 和 arm64 架构
- **完全本地化**：无外链资源依赖，可离线运行

## 🛠️ 安装方法

### 方法一：使用 Docker Compose（推荐）

1. **创建 docker-compose.yml 文件**
   ```yaml
   services:
     fluxbox:
       image: ghcr.io/kronus09/fluxbox:latest
       container_name: fluxbox
       ports:
         - "20504:20504"
       volumes:
         - ./data:/app/data
       restart: unless-stopped
       environment:
         - TZ=Asia/Shanghai
         - GIN_MODE=release
   ```

2. **启动服务**
   ```bash
   docker-compose up -d
   ```

3. **访问 Web 界面**
   打开浏览器访问：`http://localhost:20504`

### 方法二：使用 Docker 命令

```bash
docker run -d \
  --name fluxbox \
  -p 20504:20504 \
  -v $(pwd)/data:/app/data \
  --restart unless-stopped \
  -e TZ=Asia/Shanghai \
  -e GIN_MODE=release \
  ghcr.io/kronus09/fluxbox:latest
```

### 方法三：直接运行

1. **安装 Go 环境**（版本 1.21+）

2. **克隆项目**
   ```bash
   git clone https://github.com/kronus09/fluxbox.git
   cd fluxbox
   ```

3. **构建并运行**
   ```bash
   go mod tidy
   go build -o fluxbox main.go
   ./fluxbox
   ```

## 🚀 使用方法

### 1. 添加源

1. 在 Web 界面的 "添加新源" 部分填写源名称和 URL
2. 点击 "确认添加" 按钮

### 2. 同步聚合

1. 点击 "同步聚合" 按钮开始聚合过程
2. 标题栏状态栏会显示同步进度（蓝色加载状态）
3. 同步完成后，状态栏显示绿色成功提示

### 3. 生成多仓

1. 点击 "生成多仓" 按钮生成多仓配置
2. 系统会按响应速度排序，仅包含测试成功的源
3. 生成完成后可点击 "复制订阅" 获取多仓配置地址
4. 在影视Box中添加该地址，即可自由切换多个源

### 4. 聚合设置

点击标题栏的 ⚙️ 图标打开设置面板：

- **聚合模式**：
  - 最快站点：按响应时间排序，可选择前60/80/100/120/140/160/180/200个站点
  - 全部站点：聚合所有站点
  - 提示：站点数量过多可能导致搜索时并发请求过高
- **过滤关键词**：
  - 默认过滤：直播、儿童、启蒙、教育、课堂、学习、少儿、预告
  - 可自定义添加或删除关键词
  - 点击"重置默认"恢复初始设置
- **首页菜单来源**：
  - 自动选择：使用响应最快的源的菜单
  - 手动选择：可选择使用哪个源的菜单作为首页显示
  - 选择后需重新聚合生效
- **计划任务**：
  - 启用定时聚合：开启后可设置自动执行时间
  - 执行频率：每天或每周
  - 执行时间：设置具体执行时间点
  - 执行日期：选择每周模式下执行的日期（周一至周日）

### 5. 复制订阅地址

1. 点击 "复制订阅" 按钮
2. 订阅地址会自动复制到剪贴板
3. 将订阅地址添加到 影视Box 中

### 6. 管理源

- **修改源**：点击源右侧的 "修改" 按钮，在弹出的模态框中修改源信息
- **删除源**：点击源右侧的 "删除" 按钮
- **健康检查**：点击 "检查" 按钮检测源的健康状态
  - 🟢 健康：源配置正常，jar 可用
  - 🟡 一般：源配置正常，但部分 jar 不可用
  - 🔴 不健康：源配置有问题，jar 大部分不可用
  - ⚫ 失效：源无法访问
- **批量健康检查**：点击 "健康检查" 批量检测所有源

### 7. 状态提示

所有操作的状态提示统一显示在标题栏右侧的状态栏中：
- **蓝色加载状态**：操作进行中
- **绿色成功提示**：操作成功（3秒后自动恢复）
- **红色错误提示**：操作失败（显示错误信息）

## ⚙️ 配置说明

### 数据持久化

- Docker 部署：数据存储在当前目录的 `data/` 文件夹中（绑定挂载）
- 直接运行：数据存储在 `data/` 目录中

### 主要文件

- `data/sources.json`：存储源配置
- `data/config_cache.json`：存储聚合后的配置缓存
- `data/multi_config.json`：存储多仓配置
- `data/filter_words.json`：存储过滤关键词配置
- `data/schedule.json`：存储计划任务配置
- `data/home_menu_source.json`：存储首页菜单来源配置
- `data/sources/`：存储各源的独立配置缓存

### 环境变量

- `TZ`：时区设置，默认 `Asia/Shanghai`

## 🔧 常见问题

### 1. 同步失败

- 检查源 URL 是否可访问
- 检查网络连接
- 查看容器日志：`docker logs fluxbox`

### 2. 订阅地址无法访问

- 检查容器是否正常运行：`docker ps`
- 检查端口映射是否正确
- 检查防火墙设置

### 3. 数据丢失

- 确保使用了数据卷持久化
- 定期备份 `data/` 目录
